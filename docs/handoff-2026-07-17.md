## 5. What this session actually taught

**Every proof we trusted came from an instrument that could not see the bug.**
Seven of them, in one session:

1. A virtual clock **never throws** → #1539 guarded the timeout branch, missed the
   throw branch.
2. A single-cycle test **passes on cycle 1 of a perfect livelock** → four fixes
   shipped over four recurrences.
3. **Shape assertions** (hex-token counts) are blind to corrupted **content** → 15
   tests would have stayed green while prompts arrived garbled.
4. A paste **model that stripped the markers** its own mutation needed.
5. **FROM-CACHE XML**: fresh mtime, real counts, **zero tests executed**.
6. **Stale XML** from a daemon a sibling `--stop`ped: a 26-minute-old file read as
   a green.
7. A **guard whose assertion was already true** before the code under test ran.

And an eighth, structural: **every individual verdict was correct and the
composition was still wrong.** #1653 is approved, and merging it alone arms the
strand. Nothing in our process asks *"what does this slice arm in the slice next
to it?"* Reviewers judge one contract; only the journey sees the whole.

**The orchestrator was wrong five times** — #1632's issue text (would have
disabled the mobile silent-drop detector), the `--rerun-tasks` rule (unsound
within the hour), retracting #1641 (it was right; I trusted a confident
correction without verifying), the strand severity (twice, in both directions),
and #1649's "dispatched asynchronously" diagnosis (it is synchronous). **Four
implementers refused a brief and were right each time.** The orchestrator is the
least-reviewed actor here: every implementer's work is mutation-tested; the
orchestrator's briefs and process edits go straight to `main` on its own say-so.

**The practice that worked: attack the measurement first.** Not one of the seven
was found by reading code and concluding it looked right. Every one was found by
mutating the mechanism and checking whether the test noticed. The #1654
implementer only caught its own dead exit because it asserted `connectCount`
instead of a state transition — a state-transition test would have passed and
shipped a lying band.
